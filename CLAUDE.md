# CLAUDE.md — память Среды «site-factory»

> Claude читает этот файл в начале КАЖДОЙ сессии. Здесь — кто я, что за проект,
> инварианты и карта артефактов. Это не промпт: правила живут тут, а не переписываются заново.

## Кто я
Я — фабрика сайтов. Из брифа клиента собираю лендинг/сайт по управляемому
конвейеру, где каждый этап оставляет артефакт. Никакой «магии промпта» — только система.

## Стек (НЕ менять без явной команды)
- Vite + React (TypeScript)
- shadcn/ui + Tailwind
- Дизайн-токены — CSS custom properties (`tokens.css`) + W3C JSON (`tokens.json`)
- Дизайн-система по умолчанию: **Ember** — charcoal `#101010`, coral accent `#FF5050`
- Figma — через MCP (см. figma.config.json)
- Картинки/предметка/обложки — через скилл nano-banana (Gemini)

## ИНВАРИАНТЫ (нарушение = стоп и переспросить)
1. Цвета/отступы/типографика — ТОЛЬКО из токенов. Хардкод hex/px в компонентах запрещён.
2. Каждый этап пишет артефакт в artifacts/. Без артефакта этап не закрыт.
3. Этап читает артефакт предыдущего этапа, а не пересказ из чата.
4. Гипотезы помечаю тегом ГИПОТЕЗА, проверенные факты — ФАКТ.
5. Контраст по WCAG AA. Меньше — переспросить, не «улучшать молча».
6. Чистый пайплайн: без мёртвого кода, без лишних зависимостей.

## Карта артефактов (где что лежит)
- artifacts/00_brief.md        — боли, ЦА, цель, tone of voice
- artifacts/01_research.md     — конкуренты, рефы, ФАКТ/ГИПОТЕЗА
- artifacts/02_SPEC.md         — IA, список экранов/секций, контент-модель, метрики
- artifacts/03_design-system/  — STYLE_GUIDE.md, tokens.css, tokens.json, moodboard
- artifacts/04_assets/         — сгенерированные изображения + промпты
- artifacts/05_build/          — заметки сборки, карта компонентов
- artifacts/06_review.md       — контраст-аудит, токен-дисциплина, a11y, чек-лист
- HANDOFF.md                   — финальная сдача

## Конвейер (директивы в .claude/commands/)
/brief → /research → /spec → /design → /assets → /build → /review → /handoff
Оркестратор /site прогоняет цепочку и встаёт на паузу там, где нужно моё «ОК».

## Скиллы (вызываются директивами автоматически)
- style-decompose      — снять токены с референсов (пипеткой, не на глаз)
- vite-react-prototype — собрать сайт по токенам и SPEC
- nano-banana          — сгенерировать визуал/предметку
